## 1. Контракт v2 страницы «Дашборд» — `GET /api/dashboard`

**Статус: ДЕЙСТВУЕТ (04.07.2026), без переходного периода.** Версионирование
убрано решением продукта: `GET /api/dashboard` сразу отдаёт контракт v2
с реальными данными; мокап `data/dashboard.json` и v1-схема удалены.
**Страница дашборда в браузере не работает до миграции фронта на v2** —
это осознанно (прод не запускается до полной готовности).

Контракт проверен сквозным тестом 07.07.2026: временный адаптер v2->v1
поверх текущего рендера отрисовал страницу на реальных данных без ошибок
в консоли (KPI, LfL-бейджи, график, поповеры). Референс-реализация всех
производных (LfL, доли, фудкост юнитов, тексты попапов) —
`docs/mockups/dashboard-v2-adapter.reference.js`: можно взять формулы
оттуда, отображение — на ваше усмотрение.

**Принцип v2:** бэкенд отдаёт факты (числа, даты числами), фронтенд —
всё представление: форматирование сумм и валюту, названия месяцев
и падежи, проценты LfL и стрелки, доли юнитов, подписи. Прецедент —
страница sales (фронт уже считает rev/gp/fc из qty/price/unitCost).

  "units": [

    {"key": "w", "revenue": 305916,  "cost": 99325, "prevRevenue": 287100, "prevCost": 90200},

### Семантика полей

- `period` — показываемый период: месяц последнего закрытого дня,
  с 1-го числа (`dayFrom`) по последний закрытый день (`dayTo`).
- `compare` — период сравнения: те же календарные числа прошлого года
  (29 февраля в невисокосном году усекается до 28-го).
- `kpis.*.prev` — значение метрики за `compare`; LfL-процент и стрелку
  фронт считает сам: `(value - prev) / prev`.
- `checks`/`guests` — только по чекам **с выручкой**: целиком бесплатный
  чек (представительские, стафф-питание) — не продажа и не размывает
  средний чек. Гости платного чека с бесплатными позициями считаются все.
- `kpis.avgCheck` = выручка/чеки, посчитан бэком для удобства
  (в т.ч. прогнозный = forecast выручки / forecast чеков).
- `forecast` — run-rate-прогноз на конец месяца по средним РАБОЧИМ дням
  недели; регулярные выходные распознаются по 8 неделям истории
  и прогнозируются нулём, разовый простой прогноз не занижает.
- `revenueByDay` — полный календарь периода, закрытые дни заведения
  присутствуют с нулями. `weekday`: 0=вс..6=сб (совместим с
  `WEEKDAYS_SHORT` в palette.js).
- `units` — суммы по подразделениям. `key`: k/b/w + `o` («вне
  подразделений» — группы iiko вне папок Кухня/Бар/Вино). Доли
  и фудкост-проценты — производные на фронте: доля = revenue юнита /
  сумма revenue k+b+w (`o` в донат не входит, в общую выручку входит);
  фудкост = cost/revenue; дельта к прошлому году — из prev*-полей.

### Семантика null и нуля (важно!)

| Значение | Смысл | Отображение (предложение) |
|---|---|---|
| `null` | **модуля/данных ещё нет** (планы не внесены, отзывы не подключены, прогноз < 7 закрытых дней) | скрыть блок или «—» |
| `0` | **честный ноль из данных** (день закрыт, юнит без продаж, гости не введены) | показать 0 — это сигнал «проверь кассу» |

- `plan: null` — таблицы целей нет (появится с модулем targets).
- `reviews: null`, `stock: null` — до модулей отзывов и склада;
  их форма будет согласована при появлении модулей.
- `forecast: null` — в месяце меньше 7 закрытых дней.

### Что умерло из v1 (не ищите в v2)

`greeting`, `period.label/note/compareWith` (строки), `kpis.*.lfl`,
`perGuest/perCheck`, `qualityFlag`, `revenueByDay[].avg`,
`revenueByDayMax`, `categories`, `foodcostMini`, `details` —
всё это производные или представление, теперь зона фронта.

## 2. Записки по действующим контрактам

1. **`/api/sales`: `positions[].cat` получил четвёртое значение `'o'`**
   («вне подразделений»). В донат юнитов не входит; в таблице позиций,
   ABC и общей выручке участвует. Черновик имени/цвета уже добавлен
   в `palette.js` (`o: 'Вне подразделений'`, `#8A8FA3`) — примите
   или замените своим решением.
2. **`/api/sales`: `positions[].qty` — дробный** (вес: 4.5 кг).
3. **Валюта**: у пилота сомы (KGS), «₽» захардкожен в `format.js`.
   Предлагается настройка валюты на стороне фронта (конфиг инстанса).
4. **`lfl` в v1-контрактах остальных страниц**: у ресторана без прошлого
   года сравнивать не с чем — при миграции страниц на v2-стиль
   закладывайте `prev: null`.
